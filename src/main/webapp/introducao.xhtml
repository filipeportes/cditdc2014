Introdução


A especificação CDI (Context and Dependency Injection) - JSR 299 veio preencher uma importante lacuna que existia nas especificações do Java EE, em relação a injeção de dependências, seu objetivo inicialmente era prover um modelo único e simplificado de intergração entre os diversos serviços do Java EE.
Atualmente a CDI oferece diversas outras funcionalidades entre elas um modelo de eventos/notificações, escopos adicionais, interceptors, decorators, e uma SPI (service provider interface) completa que permite outros frameworks de se integrarem ao Java EE de forma simples, também conhecida como CDI Extensions.

Com esse conjunto de funcionalidades que integra todos os serviços da plataforma Java e ainda outros frameworks, CDI tem tudo para se tornar a nova ferramenta padrão para diminuir o acoplamento em projetos java.
